Monitoring TIBCO EMS Appliance V2.1 and higher

If the EMS server is running on the TIBCO EMS Appliance hardware device V2.1 (or higher) the fault tolerant state is displayed in the FaultTolerantState column on the main EMS server display, the following values can be displayed:

  • REPLICATING  This is shown in normal operation when the server is replicating with its peer and the secondary is standing by for failover.
  • STANDALONE  This is shown when the server is running standalone with no fault tolerance in operation, e.g. the second appliance is down.
  • SYNCHRONIZING(50%)  This is shown when the server is synchronizing state with its peer. The overall sync progress % is shown.
  • PAUSED  This is shown when the server is paused.
  • WAIT_FOR_PEER  This is shown when the server waiting for its peer to come up.
  • ERROR  This is shown when the fault tolerant state cannot be determined.

Note; for correct monitoring status ensure you upgrade the EMS client library to V8.1 or higher.

To show normal operating state as FULLY_OPERATIONAL (instead of REPLICATING) to be consistent with software FT state monitoring set the ShowApplFullyOp to true in the gems.props file.

Monitoring TIBCO EMS Appliance V2.0

Monitoring EMS Appliance V2.0 is not supported in this release, please use Gems 4.1.
